# Getting Started

Use these pages to learn what cuVS does, where it fits in a vector-search system, and which guide to read next.

## Introduction

- [Vector Search](choosing_and_configuring_indexes.md): learn the main index families, how they trade recall for performance, and where to start when choosing an index.
- [Clustering](clustering.md): compare K-Means, single-linkage, and spectral clustering, including why K-Means is often the preferred partitioning method for large vector search systems.
- [Vector Compression](quantization.md): compare quantization and dimensionality reduction methods for reducing vector storage, bandwidth, and search cost.
- [Vector Database](vector_databases_vs_vector_search.md): understand the difference between a vector index and a vector database, including local, global, and hybrid partitioning.
- [Tuning Indexes](tuning_guide.md): tune standalone indexes and vector databases for recall, latency, throughput, build time, and memory.
