# Getting Started

Use these pages to learn what NVIDIA cuVS does, where it fits in a vector-search system, and which guide to read next.

## Introduction

- [Vector Search](/getting-started/introduction/vector-search): learn the main index families, how they trade recall for performance, and where to start when choosing an index.
- [Clustering](/getting-started/introduction/clustering): compare clustering methods, including why K-Means is often the preferred partitioning method for large vector search systems.
- [Vector Compression](/getting-started/introduction/compression): compare quantization and dimensionality reduction methods for reducing vector storage, bandwidth, and search cost.
- [Vector Database](/getting-started/introduction/vector-database): understand the difference between a vector index and a vector database, including local, global, and hybrid partitioning.
- [Tuning Indexes](/getting-started/introduction/tuning-indexes): tune standalone indexes and vector databases for recall, latency, throughput, build time, and memory.
